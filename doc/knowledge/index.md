# 我自己的前端知识库

## :bulb: 理顺知识脉络
前端的轮子实在是太多了，从现代框架的迭代、媒体处理、性能优化、跨端应用、真机监控等等。我需要一个系统的知识地图，来我理清这些知识的脉络。这些文档不将重点放在技术细节，而是关注于知识和逻辑的整理，用更容易理解的话整理和翻译官方文档。能够用自己的话输出知识是学习的有效形式，做好官网超链接、有趣的案例记录，设想起来应该是有用而有趣的~  🤓


:::tip 🗓️ 长期坚守？
知识库有太多的形式，从大学时期使用的有道云笔记、到尝试用git项目做知识和demo结合、再到现在的vitePress；
回想毕业以来，会因为某些情况导致知识库越来越乱，打算就这次机会重新整理，希望能长期坚守下去。
:::
